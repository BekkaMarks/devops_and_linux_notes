# Entendendo GPL, Copyleft e Código Aberto

## O que é GPL (GNU General Public License)?

A **GPL (GNU General Public License)** é uma licença livre para softwares e outras obras digitais. Ela garante quatro principais liberdades para quem utiliza o programa:

- Executar o programa para qualquer finalidade.
- Estudar como o programa funciona e adaptá-lo.
- Compartilhar o programa com outras pessoas.
- Modificar e aprimorar o programa, distribuindo essas melhorias.

## Qual a relação entre GPL e Copyleft?

A GPL é um exemplo de licença que usa o conceito de **copyleft**. O **copyleft** é uma regra que garante que toda versão modificada ou programa derivado de um software original continue livre. Ou seja, se alguém modificar um programa sob GPL, é obrigatório que a nova versão também seja distribuída sob a mesma licença. Isso protege a liberdade dos usuários e impede que o software derivado se torne fechado ou proprietário.

**Resumindo:**  
A GPL usa o copyleft para obrigar que todas as futuras versões e derivações do software permaneçam livres e com o código aberto, mantendo os direitos dos usuários sempre garantidos.

## O que é código aberto (open source)?

Código aberto, ou **open source**, é um modelo de desenvolvimento onde o código-fonte do programa é disponibilizado para o público. Isso permite que qualquer pessoa:

- Veja e estude como o sistema funciona;
- Proponha melhorias ou correções;
- Colabore no desenvolvimento do projeto;
- Redistribua o programa livremente.

Em projetos open source, normalmente o código fica em um repositório público (como no GitHub), incentivando a participação da comunidade e tornando o software melhor e mais seguro com o tempo.

## Diferença entre GPL e outras licenças de código aberto

- **GPL:** Garante que todas as versões modificadas também permaneçam abertas e livres, graças ao copyleft. Se você modificar um software GPL, é obrigado a liberar a nova versão sob a mesma licença.
- **Outras licenças open source:** Também permitem estudar, modificar e compartilhar, mas podem permitir que softwares derivados sejam distribuídos sob outras licenças, inclusive fechadas.
